Hosted Pyscript stubber

Create type-stubs for MicroPython webassembly (wasm) and PyScript

Requires serve.py (not plain http.server)
  1. Select one of the options above, then click Generate stubs.
  2. The first time you load this page you will be asked to select a temp folder to sync the stubs into.
    Do not select the entire micropython-stubs folder (too many files).
  3. Wait for the stubs to be generated and synced to the local filesystem.
  4. Copy them to your project and run stubber merge & stubber build to generate the final stubs.

The generated stubs will be available in the /stubs folder in the virtual file system, and synchronised to the local filesystem. Please use the PyScript Persistent filesystem dialog to select a folder on your local filesystem where the stubs will be saved.

Note that the stubs will be saved in a subfolder stubs of the selected folder. Do not select a folder that already contains many files, as this will slow down, or completely block the synchronisation.

Local build: place micropython-webassembly-pyscript-{name}.zip files (each containing micropython.mjs and micropython.wasm) in firmware/webassembly/ and run python serve.py instead of python -m http.server.

Note: The stubs are generated using the micropython-stubber for the MicroPython stubs project.
