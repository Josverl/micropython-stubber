Hosted Pyscript stubber

Create type-stubs for MicroPython webassembly (wasm) and PyScript

  1. Set the desired MicroPython version in the createstubs-req.toml file.
  2. The first time you load this page you will be asked to select a temp folder to sync the stubs into.
    3. do not select the entire micropython-stubs folder ( too many files )

    Optionally: to select a different target location for the stubs.
  3. Wait for the stubs to be generated and synced to the local filesystem.
  4. Copy them to your project and run stubber merge & stubber build to generate the final stubs for your project.

This example shows how to create and download the MicroPython firmware-stubs for webassembly-pyscript.

The generated stubs will be available in the /stubs folder in the virtual file system, and synchronised to the local filesystem. Please use the pyscript Persisten filesystem dialog to select a folder on your local filesystem where the stubs will be saved.

Note that the stubs will be saved in a subfolder stubs of the selected folder. Do not select a folder that already contains many files, as this will slow down, or completely block the synchronisation .

Note: The stubs are generated using the micropython-stubber for the MicroPython stubs project.
